# API 参考

Ticos 平台提供了一套完整的 API，用于管理具身智能终端、处理人脸识别以及集成 LLM 功能。本文档涵盖了所有可用的 API 端点及其使用方法。

## 认证

所有 API 端点都需要使用 Bearer Token 进行认证。Token 应该是您的终端密钥，通过 Authorization 请求头传递：

```http
Authorization: Bearer <terminal_secret>
```

## 基础 URL

所有 API 端点都基于以下基础 URL：

```
https://api.ticos.ai/v1/api
```

## API 端点

### 人脸管理

#### 获取所有人脸

```http
GET /faces
```

获取与终端关联组中的所有 faces。

**响应**

```json
[
  {
    "face_id": "face123",
    "group_id": "user_id_identifier",
    "id": 39,
    "name": "测试人脸"
  }
]
```

#### 创建人脸

```http
POST /faces
Content-Type: multipart/form-data
```

在终端的关联组中创建新的人脸。

**参数**

- `image`: 人脸图像文件（必需）
- `name`: 人脸名称（必需）

**响应**

```json
{
  "return_code": 0,
  "return_message": "成功"
}
```

#### 获取特定人脸

```http
GET /faces/{face_id}
```

获取终端组中的特定人脸。

**响应**

```json
{
  "face_id": "face123",
  "group_id": "user_id_identifier",
  "id": 39,
  "name": "测试人脸"
}
```

#### 更新人脸

```http
PUT /faces/{face_id}
Content-Type: multipart/form-data
```

更新终端组中的特定人脸。

**参数**

- `name`: 人脸名称（必需）
- `image`: 人脸图像文件（可选）

**响应**

```json
{
  "return_code": 0,
  "return_message": "成功"
}
```

#### 删除人脸

```http
DELETE /faces/{face_id}
```

删除终端组中的特定人脸。

**响应**

- 204 No Content

#### 获取人脸图像

```http
GET /faces/{face_id}/image
```

获取特定人脸的图像。

**响应**

- 图像数据（适当的 MIME 类型）

### 错误响应

所有端点都可能返回以下错误响应：

- `400 Bad Request`: 请求参数无效
- `401 Unauthorized`: 认证失败
- `404 Not Found`: 资源未找到
- `500 Internal Server Error`: 服务器错误

## 实现细节

### Face ID 生成

创建人脸时，系统会自动生成唯一的 face_id。face_id 是一个 32 字符的十六进制字符串，由 16 个随机字节生成。

## 最佳实践

1. **错误处理**

   - 始终检查响应状态码
   - 为所有 API 调用实现适当的错误处理
   - 使用合适的重试机制

2. **安全性**

   - 妥善保管您的终端密钥
   - 对所有 API 调用使用 HTTPS

3. **性能**
   - 适当缓存响应
   - 实现请求批处理
   - 对大型负载使用压缩
