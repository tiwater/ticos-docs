# 姿态 (反应)

“姿态”设置位于“指令”->“反应”模块下，允许您配置您的实体代理的身体动作和姿态。这些可以增强代理的表达能力，使交互感觉更具动态性。

## 配置选项

### 1. 空闲动作

- **控件**：一个拨动开关（例如，探索时的 `ref=e90`）。
- **功能**：此设置启用或禁用代理在空闲时（即未主动执行任务或特定姿态时）进行的细微、连续的动作。空闲动作可以使代理看起来更逼真，不那么呆板。
- **示例**：如果启用，代理可能会表现出轻微的头部移动、眨眼或轻轻摇摆。

### 2. 预定义姿态

该页面通常列出代理可以执行的一组预定义姿态。在探索过程中，观察到以下姿态：

- **挥手** (例如 `ref=e96`)
  - 描述：标准的挥手姿态。
  - UI 文本键示例：`agent.config.gesture.waveHandDesc` (表示本地化描述的占位符)。
- **竖起大拇指** (例如 `ref=e107`)
  - 描述：竖起大拇指的姿态，通常表示赞同或积极。
- **指向** (例如 `ref=e118`)
  - 描述：指向的姿态，用于引导注意力。
- **鼓掌** (例如 `ref=e129`)
  - 描述：鼓掌的姿态，常用于喝彩或表达热情。

#### 对于每个预定义的姿态，通常提供以下操作：

- **校准按钮** (例如，挥手姿态的 `ref=e100`)
  - **功能**：这很可能会启动特定姿态的校准过程。校准可能包括引导代理完成动作或调整其参数，以确保虚拟形象正确、流畅地执行该姿态。
- **重置按钮** (例如，挥手姿态的 `ref=e103`)
  - **功能**：这很可能会将姿态的校准或设置恢复到其默认状态。

## 自定义姿态

- 虽然在初步探索中没有明确观察到这是一个主要功能，但平台通常会发展到允许创建或导入自定义姿态。如果存在此功能，它很可能会在本节或相关章节中进行管理。

有效使用姿态可以显著提高代理的非语言交流能力，使交互更具吸引力和直观性。
