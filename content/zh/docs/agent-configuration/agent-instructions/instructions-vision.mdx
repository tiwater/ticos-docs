# 视觉 (感知)

“视觉”设置位于“指令”->“感知”模块下，允许您配置您的实体代理如何处理和解读来自其环境的视觉信息。

## 配置选项

### 1. 核心指令 (视觉焦点)

- **描述**：虽然主要的核心指令定义了整体行为，但如果可用，视觉模块中的此部分可能用于添加特定的视觉处理准则。（在探索期间，此页面主要直接显示视觉模型和物体检测设置）。

### 2. 回应语言 (视觉上下文)

- **描述**：此下拉菜单（例如，探索时的 `ref=e88`）可能决定了基于视觉输入生成的任何文本或描述视觉元素所使用的语言。它可能与全局回应语言相关联，或在特定于视觉的任务中覆盖全局回应语言。
- **控件**：下拉菜单。

### 3. 视觉模型

- **描述**：此设置（例如 `ref=e95`）允许您选择代理用于视觉处理任务（如图像识别、场景理解或物体检测）的底层人工智能模型。
- **控件**：列出可用视觉模型的下拉菜单（例如，“GPT-4 Vision Preview”，“Gemini Pro Vision”）。
- **注意**：模型的选择会显著影响代理的视觉能力、准确性和处理速度。

### 4. 细节级别

- **描述**：此选项（例如，`ref=e104` 代表“低”，`ref=e107` 代表“高”，`ref=e110` 代表“自动”）控制代理在分析视觉输入时追求的细节级别。
- **控件**：单选按钮或选择组。
    - **低**：处理速度较快，但可能会丢失更精细的细节。
    - **高**：更详细的分析，但可能较慢且资源消耗较大。
    - **自动**：允许系统根据上下文或所选视觉模型的能力动态调整细节级别。

### 5. 物体检测

- **描述**：此部分（例如，复选框 `ref=e113`）启用或禁用代理检测和识别其视野内特定物体的能力。
- **控件**：用于切换功能的复选框。
- **物体列表**：启用后，可能会显示预定义物体的列表（例如，“人”、“汽车”、“椅子”、“桌子”、“门”、“电视”、“键盘”、“鼠标”、“手机”、“书”、“瓶子”、“杯子”、“手提包”、“笔记本电脑”、“显示器”、“遥控器”、“沙发”、“床”、“冰箱”、“烤箱”、“微波炉”、“水槽”、“烤面包机”、“时钟”、“花瓶”、“剪刀”、“泰迪熊”、“牙刷”）。在探索期间，此列表显示为禁用状态，表明它可能是未来的功能或需要特定条件才能激活。
- **功能**：如果激活，您或许可以选择代理应特别寻找或关注哪些物体。

适当配置这些视觉设置可确保您的代理能够根据其任务要求有效地“看见”和理解其环境。
