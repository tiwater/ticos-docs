# 代理技能 (工具)

代理配置中的“技能”模块允许您管理您的实体代理可以用来执行任务、与外部系统交互或以高级方式处理信息的特定能力、工具或功能。

**关于术语的说明**：在最初的平台探索中，此部分曾预期为“工具”。“技能”页面 (`/en/agents/33/skills`) 的存在表明这可能是管理此类功能的指定区域。“技能”是否直接取代或等同于“工具”尚待确认 (参考 MEMORY[eb2b0bab-3efa-4e91-a558-c298f4c2d3ba])。

## 总览

当您导航到“技能”部分时，通常会看到：

- **页面标题**：“技能”
- **内容区域**：此区域将列出已添加到代理的任何技能。如果不存在任何技能，则可能会显示相应的提示消息。
- **主要操作**：用于添加新技能的按钮。

## 添加技能

该界面提供了几种向您的代理添加技能的方法：

1.  **创建新技能按钮** (例如，探索时的 `ref=e151`)
    -   **功能**：此选项可能会打开一个对话框或界面，用于从头开始定义新技能。这可能涉及指定技能的名称、描述、输入/输出参数，甚至可能编写其逻辑或将其连接到 API 端点。

2.  **从库中添加按钮** (例如，探索时的 `ref=e155`)
    -   **功能**：这表明可以访问预先存在的技能库。您可以浏览或搜索此库，并选择要添加到代理的技能。这些可能是平台提供的或由其他用户共享的通用、可重用技能。

3.  **从 MCP 服务器添加按钮** (例如，探索时的 `ref=e159`)
    -   **功能**：MCP (模型上下文协议) 服务器通过提供对专门功能或外部信息的访问来扩展代理能力。此选项将允许您将代理连接到 MCP 服务器并集成其上托管的技能。

## 管理现有技能

添加技能后，此部分通常还会提供以下工具：

-   查看已配置技能的详细信息。
-   为代理启用或禁用特定技能。
-   配置单个技能的参数或设置。
-   移除不再需要的技能。

(管理现有技能的具体用户界面以及添加/配置每种类型技能的详细过程，在添加技能后需要进一步探索。)

为您的代理配备正确的技能组合对于使其能够执行广泛的任务并与用户和系统有效交互至关重要。
