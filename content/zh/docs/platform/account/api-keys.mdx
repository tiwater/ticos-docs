# API 密钥

API 密钥页面允许您为您的 Ticos 账户生成和管理 API 密钥。这些密钥支持对 Ticos 平台服务的编程访问，使您能够将 Ticos 功能集成到您自己的应用程序、脚本或自动化工作流程中。

## 访问 API 密钥页面

要访问 API 密钥管理页面：

1.  点击平台界面右上角的用户头像或个人资料图标。
2.  从下拉菜单中选择“账户设置”或类似选项。
3.  在账户设置中，导航至“API密钥”部分（例如，通过侧边栏链接 `ref=e59`）。

## 管理 API 密钥

主界面（例如，`ref=e61`）提供以下功能：

### 1. 创建新 API 密钥

-   点击 **“创建新API密钥”** 按钮（例如，`ref=e66`）以生成新的密钥。
-   通常系统会提示您为 API 密钥指定一个描述性名称，以帮助您识别其用途（例如，“我的应用集成”、“夜间批处理脚本”）。
-   创建后，新的 API 密钥将会显示。**重要提示：这通常是完整密钥唯一一次显示的机会。请立即复制并将其存储在安全的位置。**

### 2. 查看现有 API 密钥

标题为“您的API密钥”的部分（例如，`ref=e69`）将列出您已生成的所有 API 密钥。对于每个密钥，您通常可以看到：

-   **名称**：您为密钥分配的描述性名称。
-   **密钥片段**：密钥的一部分（例如，前几个或后几个字符）用于识别，但出于安全原因不会显示完整密钥。
-   **创建日期**：密钥生成的日期和时间。
-   **最后使用**：密钥最后一次用于进行 API 调用的日期和时间（如果跟踪此信息）。
-   **状态**：密钥是活动的还是已撤销。
-   **操作**：管理密钥的选项，例如：
    -   **撤销/删除**：永久禁用并删除 API 密钥。此操作无法撤销。
    -   **编辑名称（可能）**：某些系统允许您更改密钥的描述性名称。

*（注意：页面目前在“您的API密钥”部分显示“加载中...” (`ref=e71`)，表明密钥列表在填充后会显示在此处。）*

## 安全最佳实践

-   **像对待密码一样对待 API 密钥**：确保其机密性和安全性。
-   **不要将 API 密钥直接嵌入客户端代码中**或将其提交到版本控制系统（例如 Git）。
-   **使用环境变量**或安全的密钥管理服务在您的应用程序中存储和访问 API 密钥。
-   **为不同的应用程序或服务创建单独的 API 密钥**，以便在一个密钥泄露时限制访问范围。
-   **定期审查并撤销未使用的 API 密钥**，以最小化潜在的攻击面。
-   如果平台支持密钥的细粒度权限设置，请**仅授予 API 密钥必要的权限**（此功能可能不在此特定页面上，但属于通用最佳实践）。
