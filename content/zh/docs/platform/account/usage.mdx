# 使用情况

“使用情况”页面提供了关于您消耗 Ticos 平台资源（特别是 API 使用和其他计量服务）的洞察。这有助于您跟踪活动、理解账单并优化您的应用程序。

该页面的主要目的是“查看您的 API 使用指标”。

## 访问使用情况页面

要访问使用情况页面：

1.  点击平台界面右上角的用户头像或个人资料图标。
2.  从下拉菜单中选择“账户设置”或类似选项。
3.  在账户设置中，导航至“使用情况”部分（例如，通过侧边栏链接 `ref=e53`）。

## 主要功能（预期与观察所得）

根据典型的用量仪表盘和当前页面元素，您可以预期以下功能：

-   **日期范围选择**：一个日期选择器（例如 `ref=e67`，当前显示为 "Jun 01, 2025 - Jun 20, 2025"）允许您选择特定时间段以查看使用数据。这对于分析趋势或检查特定计费周期的消耗至关重要。
-   **API 调用指标**：关于您 API 调用的详细统计信息，可能包括：
    -   API 请求总数。
    -   按 API 端点或服务划分的调用明细。
    -   成功与失败的请求。
    -   数据传输量（流入/流出）。
-   **资源消耗**：其他计费资源的信息，例如：
    -   已用存储空间（例如，用于知识库、智能体配置）。
    -   活动终端时长。
    -   模型推理时间或消耗的令牌数。
-   **图形化表示**：使用数据通常以图表形式呈现，以便轻松可视化趋势和模式。
-   **数据导出（可能）**：某些平台允许您以 CSV 等格式导出使用数据，用于离线分析或记录保存。

## 理解您的使用情况

定期查看您的使用统计可以帮助您：

-   **监控成本**：如果您的使用情况与定价计划相关联，请跟踪您的支出。
-   **识别趋势**：了解您的应用程序对 Ticos 服务的使用如何随时间变化。
-   **优化性能**：找出频繁使用或资源密集型的 API 调用，以便进行优化。
-   **容量规划**：根据历史使用模式预测未来需求。

## 注意事项

-   可用的具体指标可能取决于您的订阅级别和您在 Ticos 平台上使用的服务。
-   报告使用数据可能存在延迟，因此显示的统计信息可能并非精确到秒的实时数据。
-   如果您对使用情况或账单有疑问，请参阅平台的定价页面或联系支持。
