# 发布变更

智能体配置中的“发布变更”模块是您审查和部署对具身智能体设置所做任何修改的地方。这是确保您的更改生效并应用于智能体的实时或活动版本的关键步骤。

## 主要功能和操作

### 1. 暂存更改状态

- **显示**：页面将指示是否存在任何已进行但尚未发布的待处理更改。
- **示例消息**：“没有要发布的暂存更改”（如果没有待处理的修改）。

### 2. 主要操作按钮

- **编辑原始配置按钮**：
    - **功能**：此选项会打开一个界面，允许您以原始 JSON 格式查看并直接编辑智能体的完整配置。这是一项高级功能，应谨慎使用。

- **重新加载智能体按钮**：
    - **功能**：此操作会重新加载智能体的当前配置，用于在智能体与数据库不同步时刷新其状态（例如在多台设备上同时编辑智能体配置时），这个操作将放弃当前未保存的临时更改。

### 3. 发布和恢复操作

这些按钮是发布工作流程的核心，通常仅在存在暂存更改时启用。

- **撤销更改按钮**：
    - **功能**：如果存在暂存更改，此按钮将允许您放弃这些更改，并将智能体的配置恢复到上次发布的状态。

- **发布更改按钮**：
    - **功能**：如果存在暂存更改，您将点击此按钮以将这些更改应用于活动智能体。此操作使修改生效。

## 工作流程

1.  在各种智能体配置模块（指令、技能、知识库等）中进行所需的更改。
2.  导航到“发布更改”页面。
3.  审查暂存更改。
4.  如果需要，可选择使用“编辑原始配置”进行高级调整，或使用“重新加载代理”。
5.  如果对更改满意，请单击“发布更改”以使其生效。
6.  如果您希望放弃更改，请单击“撤销更改”。

及时发布您的更改可确保您的智能体以最新的配置运行。
