# 发布更改

智能体配置中的“发布”模块是您审查和部署对具身智能体设置所做任何修改的地方。这是确保您的更改生效并应用于智能体的实时或活动版本的关键步骤。

## 总览

当您导航到“发布更改”页面（例如 `/en/agents/33/publish`）时，通常会看到：

- **页面标题**：“发布更改”
- **副标题/描述**：“审查并发布您已暂存的更改，或恢复到原始配置。”
- **智能体标识符**：正在审查其更改的智能体的名称（例如，“测试智能体”）。

## 主要功能和操作

### 1. 暂存更改状态

- **显示**：页面将指示是否存在任何已进行但尚未发布的待处理更改。
- **示例消息**：“没有要发布的暂存更改”（如果没有待处理的修改）。
- **如果存在更改**：通常会列出或总结在不同配置模块中所做的修改。

### 2. 主要操作按钮

- **编辑原始配置按钮**（例如，探索时的 `ref=e319`）：
    - **功能**：此选项可能会打开一个界面（可能是 JSON 或 YAML 编辑器），允许您以原始格式查看并直接编辑智能体的完整配置。这是一项高级功能，应谨慎使用。

- **重新加载智能体按钮**（例如，探索时的 `ref=e325`）：
    - **功能**：此操作可能会重新加载智能体的当前配置，可能用于应用微小更改或在智能体似乎无响应或不同步时刷新其状态。确切的行为（例如，是应用暂存更改还是重新加载上次发布的版本）需要进一步明确。

### 3. 发布和恢复操作

这些按钮是发布工作流程的核心，通常仅在存在暂存更改时启用。

- **恢复更改按钮**（例如 `ref=e337` - 在没有暂存更改时观察到为禁用状态）：
    - **功能**：如果存在暂存更改，此按钮将允许您放弃这些更改，并将智能体的配置恢复到上次发布的状态。

- **发布更改按钮**（例如 `ref=e337` - 在没有暂存更改时观察到为禁用状态）：
    - **功能**：如果存在暂存更改，您将点击此按钮以将这些更改应用于活动智能体。此操作使修改生效。

## 工作流程

1.  在各种代理配置模块（指令、技能、知识库等）中进行所需的更改。
2.  导航到“发布更改”页面。
3.  审查暂存更改的摘要（如果可用）。
4.  如果需要，可选择使用“编辑原始配置”进行高级调整，或使用“重新加载代理”。
5.  如果对更改满意，请单击“发布更改”以使其生效。
6.  如果您希望放弃更改，请单击“恢复更改”。

定期发布您的更改可确保您的代理以最新的配置运行。
