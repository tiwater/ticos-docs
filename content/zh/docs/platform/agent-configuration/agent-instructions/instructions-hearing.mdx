# 听觉 (感知)

“听觉”设置位于“指令”->“感知”模块下，允许您配置您的具身智能体如何处理来自其环境的音频输入，尤其侧重于语音检测。

## 配置选项

### 1. 语音检测

本节重点介绍如何配置智能体检测人声。

- **标题**：语音检测
- **描述**：配置语音检测方法及相关参数。

#### a. 语音检测方法

- **标签**：语音检测方法
- **控件**：一个拨动开关（例如，探索时的 `ref=e199`）。
- **功能**：此开关可能允许您在不同的语音检测机制之间进行选择。
- **提示信息**：“默认为终端侧检测，启用以使用服务器侧检测来实现对话存在感。” 这表明存在两种主要模式：
    - **终端侧检测 (默认)**：语音处理在智能体运行的终端或设备上本地进行。这样可以更快并保护隐私，因为音频数据可能不需要发送到服务器。
    - **服务器侧检测 (启用时)**：语音处理由服务器处理。这可能提供更强大的检测能力，或者对于某些功能（如连续对话存在感检测）是必需的。

### 2. 输入设备

本节涉及指定音频输入源。

- **标题**：输入设备

#### a. 输入设备选择

- **标签**：输入设备
- **控件**：一个下拉菜单（组合框，例如，探索时的 `ref=e208`）。
- **功能**：允许您选择代理应用于监听环境的特定麦克风或音频输入通道。
- **示例值**：在探索期间，显示了“PCM 16-bit”，这是一种常见的音频格式。下拉菜单通常会列出系统识别的可用音频输入设备。

对于需要响应语音命令、参与对话或对环境中特定声音做出反应的代理来说，正确配置这些听觉设置至关重要。
