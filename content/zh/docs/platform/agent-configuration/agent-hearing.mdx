# 听觉 (感知)

“听觉”模块用于配置智能体处理音频输入的方式，其核心是语音检测功能。

## 配置选项

### 语音检测

本节用于配置智能体检测人声的方法及相关参数。

- **语音检测方法**: 此开关用于选择语音检测的执行位置。系统提供两种模式：
    - **终端侧检测 (默认)**: 语音处理在设备本地进行。此模式响应速度更快，且能较好控制数据流量，因为无效的音频数据无需上传至服务器。
    - **服务器侧检测**: 启用后，语音处理将由云端服务器执行。此模式通常识别能力更强，但对数据流量会消耗更多。启用后，您可以进一步配置以下参数：
    - **灵敏度**: 用于调整语音检测的敏感程度。值越低，检测越灵敏，但也可能更容易将背景噪音误判为语音。例如，`0.05` 是一个相对灵敏的设置。
    - **语音前置缓冲 (毫秒)**: 在检测到语音开始前，系统捕获并保留的音频时长。这有助于确保即使在语音开始的瞬间也能完整识别，避免丢失开头的词语。
    - **静音时长 (毫秒)**: 判断一句话结束所需的静音时间。当系统检测到超过此时长的静音时，会认为用户已经讲完话，并将该段音频发送进行识别。

### 输入设备

本节用于指定音频输入的来源。

- **输入设备**: 此下拉菜单用于选择智能体监听环境所使用的音频设备，例如特定的麦克风。菜单中会列出系统当前识别的所有可用音频输入通道。此选项目前只对 PC 端测试有效。

通过正确配置这些听觉设置，可以确保您的智能体能够灵敏地响应语音指令，并流畅地参与对话。
